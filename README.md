# Telegram-бот для поздравлений менторов Devman

## Описание

Этот бот помогает автоматически поздравлять менторов с праздниками, предоставляя удобный интерфейс для выбора ментора и отправки персонализированной открытки. Сделайте поздравления более удобными и эффективными!

## Структура проекта
### Основные файлы и директории:
- `main.py:` Главный файл для запуска Telegram-бота. Настраивает обработчики команд и сообщений.
- `data/process_response_data.py:` Модуль для взаимодействия с API. Содержит функции для получения и валидации данных.
- `data/models.py:` Определяет Pydantic-модели данных для менторов, открыток и праздников.
- `handlers/handlers.py:` Обработчики событий Telegram-бота, включая команды и сообщения.
- `utils/format_long_name.py:` Утилита для форматирования длинных имен менторов.
- `utils/start_role.py:` Определяет роль пользователя (ментор или ученик) на основе его Telegram ID.
- `utils/insert_name.py:` Вставляет имя в открыту поздравления.
- `utils/long_message.py:` Обрезает сообщения, которые более 4096 длинной.

## Установка
1. Клонируйте репозиторий:

    ```python
    git clone <URL репозитория>
    cd <имя директории>
    ```
2. Установите зависимости:
    ```python
    pip install -r requeriments.txt
    ```
3. Создайте файл .env в корне проекта и добавьте ваш Telegram токен:
    ```text
    TG_TOKEN=ваш_токен_бота
    ```
4. Убедитесь, что локальный сервер FastAPI запущен для работы с API:
    ```python
    uvicorn api_url:app --host 127.0.0.1 --port 8000 --reload
    ```

## Запуск
Для запуска Telegram-бота выполните:
    ```
    python main.py
    ```

## Функциональность бота
1. `Команда /start:`

    - Приветствует пользователя и определяет его роль (ментор или ученик).

    - Показывает клавиатуру с кнопками:
        - "Показать менторов"
        - "Показать открытки"
        - "Показать праздники"

2. `Кнопки:`

    - "Показать менторов": Отображает список менторов с их именами и ссылками на Telegram.
    - "Показать открытки": Показывает список доступных открыток.
    - "Показать праздники": Отображает список праздников.

3. `Выбор ментора и отправка открытки:`

    - Пользователь выбирает ментора из списка.
    - Затем выбирает открытку, которая будет отправлена выбранному ментору.

4. `Обработка ошибок:`

    - Обрабатываются ошибки Telegram API (например, пользователь заблокировал бота).
    - Предоставляются сообщения об ошибках при некорректных действиях.



## Локальный сервер API для тестирования

Для тестирования бота с вашими собственными данными и схемами, вы можете развернуть локальный сервер API.

### Шаги по развертыванию локального сервера

1.  **Установите FastAPI и Uvicorn:**

    ```
    pip install fastapi uvicorn
    ```

2.  **Создайте файл `ваше_название.py` с вашим API:**

    Создайте файл `ваше_название.py` (или любое другое имя по вашему выбору) и определите ваши API endpoints, используя FastAPI.  Например:

    ```
    from fastapi import FastAPI, HTTPException
    from pydantic import BaseModel
    from typing import List, Dict

    app = FastAPI()

    class Mentor(BaseModel):
        id: int
        name: Dict[str, str]
        tg_username: str
        tg_chat_id: int

    class Postcard(BaseModel):
        id: int
        holidayId: str
        name_ru: str
        body: str

    class Holiday(BaseModel):
        id: int
        name_ru: str
        date: str

    class ResponseData(BaseModel):
        mentors: List[Mentor]
        postcards: List[Postcard]
        holidays: List[Holiday]

    # Пример данных (замените своими)
    mentors_data = [
        {"id": 1, "name": {"first": "Иван", "second": "Devman"}, "tg_username": "@ivan", "tg_chat_id": 2321},
        {"id": 2, "name": {"first": "Коля", "second": "Devman"}, "tg_username": "@коля", "tg_chat_id": 32}
    ]

    postcards_data = [
        {"id": 1, "holidayId": "01.01", "name_ru": "Новый год", "body": "🎉✨С Новым годом!✨🎉"}
    ]

    holidays_data = [
        {"id": 1, "name_ru": "Новый год", "date": "01.01"}
    ]

    @app.get("/api/v1/mentors", response_model=List[Mentor])
    async def get_mentors():
        return mentors_data

    @app.get("/api/v1/postcards", response_model=List[Postcard])
    async def get_postcards():
        return postcards_data

    @app.get("/api/v1/holidays", response_model=List[Holiday])
    async def get_holidays():
        return holidays_data
    ```

3.  **Запустите FastAPI сервер:**

    ```
    uvicorn main:app --host 127.0.0.1 --port 8001 --reload
    ```

4.  **Измените URL в `data/process_response_data.py`:**

    Закомментируйте или удалите другие URL и убедитесь, что активный URL указывает на ваш локальный сервер:

    ```
    url = 'http://127.0.0.1:8001/api/v1/mentors'
    ```

Теперь ваш бот будет использовать данные с вашего локального сервера API.


## Пример использования на тестовых данных
1. Валидные данные:
    - Валидный JSON, соответствующий ожидаемой схеме.
    - Используется для проверки корректной работы приложения при получении данных менторов, открыток и праздников.

    ```json

        ]
    },
    "postcards": {
        "postcards": [
        {"id": 1, "holidayId": "01.01", "name_ru": "Новый год", "body": "🎉✨С Новым годом!✨🎉"}
        ]
    }
    }```
2. Невалидные данные:
    - Невалидный JSON (например, отсутствует запятая или ключи добавлены неправильно).
    - Используется для проверки обработки ошибок при работе с некорректными данными.

    ```json
    {

        ]
    }
    }```
3. Данные с неправильной схемой:
    - Содержит JSON, который не соответствует ожидаемой схеме API.
    - Например, ключи или структура данных отличаются от ожидаемых.
    - Используется для проверки валидации данных и обработки случаев, когда структура данных неожиданно изменяется.
    
    ```json
    {
    "mentors_list": [
        {"id": 1, "name_first": "Евгений", ...} // Ключи не соответствуют ожидаемой схеме
    ]
    }```

## Обработка ошибок
Приложение обрабатывает различные типы ошибок для обеспечения стабильности и информативности. Ниже приведены основные типы ошибок и их описание:

1. Сетевые ошибки `(NetworkError)`
Эти ошибки возникают при проблемах с сетевым соединением, например, если сервер недоступен или нет соединения с Интернетом.

2. Ошибки формата ответа `(ResponseFormatError)`
Эти ошибки возникают, когда ответ от сервера имеет неожиданный формат, что может привести к проблемам с парсингом данных.

3. Ошибки сервера `(ServerError)`
Эти ошибки включают в себя различные проблемы на стороне сервера, такие как невалидный JSON, неправильные HTTP-запросы или ошибки аутентификации.

4. `Общие ошибки`
Любые другие необработанные исключения также обрабатываются и регистрируются для дальнейшего анализа.

Примеры ошибок
`JSONDecodeError`: Возникает при невалидном JSON в ответе от сервера.

`httpx.RequestError`: Обработка ошибок запросов, включая соединения и таймауты.

`httpx.HTTPStatusError`: Обработка HTTP-статусов, отличных от 2xx, таких как 404 или 500.
